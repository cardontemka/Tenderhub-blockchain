# TenderHub — Тендерийн блокчейн систем (v2)

Тендерийн (худалдан авах ажиллагааны) үйл явцыг блокчейн дээр **ил тод, шударга**
явуулах систем. Гэрээний санхүүжилтийг токен болгож, ажил дуусахад банкны **Escrow**
данснаас мөнгө болгон солих зарчмаар ажиллана.

- **Smart contract:** Solidity + Hardhat (`contracts/TenderHub.sol`)
- **Token стандарт:** ERC-1155 (санхүүжилт + барьцаа + хөлс + алданги + баримтын NFT)
- **Frontend:** Next.js туршилтын орчин (`frontend/`) — Role сольж, банкны симуляцтай

---

## 1. Архитектур

Тендер бүрт **нэг `TenderHub` гэрээ** Захиалагчаар deploy хийгдэнэ. Энэ нь ERC-1155
бөгөөд хоёр төрлийн токен агуулна:

| Төрөл | ID муж | Тайлбар |
|------|--------|---------|
| **Fungible санхүүжилт** | `1, 2, 3 …` | Багц (tranche) бүр — **Урьдчилгаа / Шатлал / Барьцаа / Арбитрын хөлс / Алданги** — өвөрмөц ID-тай. **1 токен = 1 ₮**. Escrow данснаас мөнгө авах эрх. |
| **Баримт NFT** | `≥ 1,000,000,000` | Гэрээ/тайлан/гомдол/тогтоол/нөхцөл/өөрчлөлт бүр `{hash, uri, timestamp, submitter}`-тэй 1ш токен. |

### Токений төлвүүд
- **Идэвхтэй / Идэвхгүй** → `frozen` (арбитр дуудах үед царцана: гүйлгээ/солилт хаагдаж, **эцсийн хугацааны тоолол зогсоно** — frozen хугацаагаар deadline урагшилна).
- **Эрхтэй / Эрхгүй** → tranche бүрийн `entitled` (шат батлагдахад мөнгө болох эрхтэй болно).
- **Хүлээгдэж буй (pending swap)** → `redeem` дуудсаны дараа токен түгжигдэж, зөвхөн банк солих боломжтой.
- **Төлөгдсөн** → солих үед **шатаагдана** (burn).

### v2.1-ийн гол дүрмүүд
1. **Хугацаа бүгд ӨДРӨӨР** тохирно (гэрээ цаанаа секундээр). **0 хоногийн эцсийн хугацаа хориотой.**
2. **Урьдчилгаа** нь зөвхөн анхны "Санхүүжилт зарлах" үед тавигдаж, дараа **өөрчлөгдөхгүй**; идэвхжих үед шууд эрхтэй болно. Урьдчилгаа + Шатлалууд = Нийт санхүүжилт.
3. **Баталгаат хугацаа** (warranty)-г мөн санхүүжилт зарлах үед өгнө.
4. **Барьцаа = нийт санхүүжилтийн яг 1%.** Гүйцэтгэгчийн мөнгө боловч **Захиалагч эзэмшинэ** ба **шилжүүлэх боломжгүй** (Escrow доод нөөц). Гэрээ цуцлагдвал/амжилтгүй болвол Захиалагч мөнгө болгож хохирлоо нөхнө. Амжилттай дуусаж **баталгаат хугацаа өнгөрсний дараа Гүйцэтгэгч рүү шилжиж** мөнгө болох эрхтэй болно (`releaseWarranty`).
5. **Дансны үлдэгдэл хүрэлцэхгүй бол мөнгө тушаах боломжгүй** (frontend банк шалгана). Цорын ганц зээл = Escrow токен солих үед хүрэлцэхгүй бол **Захиалагчийн нэр дээр** гарна.
6. **Алданги = нийт санхүүжилтийн 0.02%/өдөр**. `assessPenalties()`-г хэн ч (UI/банк автоматаар) дуудаж тооцуулна.
   - *Гүйцэтгэгч буруутай* (хугацаа хэтрүүлсэн, ажлаа өгөөгүй) → одоогийн шатнаас алдангийг **шатааж**, тэнцэхийг Захиалагчид. Ажлаа өгсний дараа буруу Захиалагч руу шилжинэ.
   - *Захиалагч буруутай* → Гүйцэтгэгчид Алданги-токен. Зөвхөн **хугацаа хэтрэлтийн** алданги одоогийн шатны **талаас** хэтэрвэл гэрээ **Suspended**.
7. **Арбитр заавал урьдчилан томилогдсон** байж гэрээ ИДЭВХЖИНЭ. Арбитр гэрээний хэсэг болсны дараа **зөвхөн давж заалдах**-аар л солигдоно.
8. **Ажлыг зөвшөөрөх/татгалзах хоёуланд нь шалтгааны NFT** хавсаргана. Татгалзвал нөгөө тал **хүлээн зөвшөөрвөл** дахин хүсэлт гаргана, эс зөвшөөрвөл **Арбитр дуудна** (хязгааргүй давтаж болно — auto-suspend байхгүй). Арбитр дуудах товч нь татгалзлыг хүлээн зөвшөөрөх товчны хажууд байрлана.
9. **Гэрээний өөрчлөлт:** **аль ч тал** (Гүйцэтгэгч ч мөн) шалтгаан + өөрчлөлтийг нэг дор санал тавина; санал тавьсан тал өөрөө зөвшөөрсөнд тооцогдох тул **нөгөө тал л баталбал** хэрэгжинэ. (Тусдаа "санал болгох" алхам байхгүй.)
9a. **Ажлын татгалзал** шийдэгдээгүй бол **өөрчлөлт оруулж болохгүй**, мөн **өөрчлөлтийн татгалзал** шийдэгдээгүй бол **ажил хүлээлгэж болохгүй**.
10. **Арбитр дуудахдаа асуудлаа СОНГОХГҮЙ** — дуудагчийн эсэргүүцэж буй татгалзлаас (ажил/өөрчлөлт/цуцлалт) автоматаар тодорхойлогдоно. Шийдвэр зөвхөн тэр асуудлыг шийднэ; ялагдсан тал хөлсийг торгууль болгон төлөх ба маргаан дуусахад **гэрээ дахин идэвхтэй болно**. Цуцлалтын шийдвэр эцсийн (давж заалдахгүй).
11. **Давж заалдах:** зөвхөн **ялагдсан тал** (зөв гэж үзсэн тал биш) давж заалдана. Шинэ арбитрыг **давж заалдсан тал нэрлэх ба түүний ЭСРЭГ тал нь батална** (арбитрыг нэрлэсэн тал өөрөө батлах боломжгүй). Хөлсийг нь **давж заалдсан тал өөрөө төлнө**.

### Банк (Escrow) — симуляц
Банк гэрээнд **албан ёсны хаягаар оролцохгүй**, зөвхөн бүртгэлтэй `oracle` хаягаар л
автоматаар хариу өгнө (худал мэдээлэл хүлээж авахгүй). Frontend дотор:
- Талууд бүгд **100,000 ₮**-тэй эхэлнэ (`lib/bank.js`, гинжин event-ээс автоматаар тооцно).
- **Дансны үлдэгдэл хүрэлцэхгүй бол** мөнгө тушаах үйлдэл (барьцаа, Escrow санхүүжилт,
  арбитрын хөлс) **боломжгүй** — яг л энгийн банк шиг. Дансны дугаар/зөвшөөрөл нь анхнаасаа
  off-chain (MetaMask/банк) тохирсон гэж үзнэ (гэрээнд хадгалахгүй).
- Банк автоматаар: алданги тооцуулах, тогтоол эцэслэх (7 хоног), солих хүсэлтийг гүйцэтгэх,
  баталгаат хугацааны дараа барьцаа буцаах.
- **Цорын ганц зээл** = Escrow токен солих үед хүрэлцэхгүй бол банк **тэр дор нь**
  **Захиалагчийн нэр дээр** зээл гарган мөнгөөр сольж өгнө (хүлээлгэгүй). Зээлийн хүү/хугацаа
  нь off-chain тооцогдоно. Талуудын хувийн данстай хамаагүй.
- Энэ зээл нь **Escrow-ийн өр** (`escrowDebt`) болж бүртгэгдэж тойм самбарт харагдана.
  Захиалагч `fundEscrow`-оор санхүүжүүлэх үед тушаасан хэмжээгээр нь **өр эхлээд дарагдаж**,
  үлдсэн нь л зарцуулагдах үлдэгдэл болно.

### NFT баримтын бүрэн бүтэц (hash баталгаажуулалт)
- Илгээсэн материал (тайлан/гомдол/тогтоол/нөхцөл) бүрийн **keccak256 hash нь blockchain
  дээр**, харин жинхэнэ контент (URL → JSON) нь **off-chain localStorage**-д хадгалагдана
  (`lib/docstore.js`, бодит DB-ийн оронд).
- Тойм самбарын **Баримт бичгүүд** хэсэгт баримт бүрийг **"Шалгах"** товчоор нээж: off-chain
  контентын hash-ийг дахин тооцож, on-chain hash-тай **тааруулна** (✓/✗).
- Контентыг өөрчилж туршихад (тест товч) дахин тооцсон hash зөрж, **"✗ Зөрчилтэй —
  контент өөрчлөгдсөн"** гэж шууд илрүүлнэ.

## 2. Функцийн зураглал (даалгавар → код)

| Workflow | Гэрээний функц | Хэн |
|----------|----------------|-----|
| 1 Deploy | `constructor` | Захиалагч |
| 3 Гүйцэтгэгч урих | `inviteContractor` | Захиалагч |
| 4 Гэрээнд орох | `joinContract` | Гүйцэтгэгч |
| 2 Санхүүжилт зарлах | `declareFunding(advance, amounts, days, warranty)` | Захиалагч |
| 4 Барьцаа(1%) + идэвхжүүлэх | `activateContract` (арбитр шаардлагатай) | Гүйцэтгэгч |
| — Escrow санхүүжүүлэх | `fundEscrow` | Захиалагч |
| 5/6/7/21 Арбитр томилох | `inviteArbiter`/`approveArbiter`/`arbiterAccept` | бүгд |
| 8–11 Зээл | `requestLoan(lender,amount)`/`offerLoan`/`acceptLoan`/`approveLoanTransfer` | Гүйцэтгэгч/Зээлдүүлэгч/Захиалагч |
| 15 Ажил хүлээлгэх | `submitDelivery` (+`acceptDeliveryRejection`) | Гүйцэтгэгч |
| 16 Батлах / Татгалзах | `approveDelivery(reason)` / `rejectDelivery(reason)` | Захиалагч |
| 17/24 Арбитр дуудах | `callArbiter(fee, uri, hash)` (асуудал авто) | Тал |
| 18 Шийдвэр (+авто эцэслэх) | `issueRuling` / `finalizeRuling` | Арбитр / банк |
| 19 Хүлээн зөвшөөрөх | `respondToRuling` | Тал |
| 20 Давж заалдах | `appeal` (цуцлалтад үгүй) | Тал |
| 22 Өөрчлөлт санал тавих | `stageAmendment(reason, …)` | аль ч тал |
| 23 Зөвшөөрөх | `respondToAmendment` (+`acceptAmendmentRejection`) | нөгөө тал |
| Цуцлах | `requestCancellation` / `respondCancellation` | Тал |
| Барьцаа буцаах | `releaseWarranty` (баталгаат хугацааны дараа) | хэн ч / банк |
| 25 Токен солих | `redeem` + `settleRedemption`(банк) | Эзэмшигч / oracle |
| Алданги | `assessPenalties` | хэн ч / банк |

## 3. Эхлүүлэх

```bash
npm install
npx hardhat compile
npx hardhat test          # 31 тест (урьдчилгаа, барьцаа/баталгаат хугацаа, алданги, зээл, маргаан, цуцлах, Escrow өр)
```

### Frontend (visual туршилт)

```bash
npx hardhat node                       # терминал A — локал блокчейн
cd frontend && npm install && npm run dev   # терминал B — http://localhost:3000
```

Зүүн самбараас **хэн болж нэвтрэхээ** сонгоно. **#0 (Захиалагч)** deploy хийнэ →
`#1` Гүйцэтгэгч орж барьцаа тушаана → ажил/санхүүжилт/маргаан/өөрчлөлтийг туршина.
Банк (`#3`) автоматаар ажиллана (солилт, алданги, тогтоол эцэслэх).

> Бодит банкны API холбохгүй — локал симуляцид key/id/url хэрэггүй. Хожим бодит банк
> холбохыг хүсвэл: API base URL, key/secret, дансны формат, redemption callback хэрэгтэй.
